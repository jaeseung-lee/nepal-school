- 도구: Playwright(Chromium, headless) 커스텀 스크립트. `networkidle` 대기 + 홈페이지는 추가 1.8초 대기(히어로 슬라이드쇼 안정화) 후 캡처

  - 필수 산출물 12개: `{home,services,about,partners,why,contact}-{desktop,mobile}.png` (풀페이지)
  - 보조 산출물 24개(분석용): `_fold_*-{desktop,mobile}.png` (뷰포트만, 스크롤 없이 보이는 영역 확인용), `_footer_*-{desktop,mobile}.png` (푸터 크롭, 페이지 간 일관성 확인용)
  - 총 36개 파일이 screenshots 디렉터리에 저장됨

## 1. Above-the-Fold 분석 (스크롤 없이 보이는 영역)

### 홈페이지 — 매우 우수
- **데스크톱(1280×800)**: "GLOBAL HUMAN RESOURCE DEVELOPMENT" 아이브로우, H1 "We Don't Send Talent. We Grow It.", 국문 서브헤드 "사람을 키우는 일이, 가장 확실한 연결입니다.", 설명 문단, CTA 버튼 2개("외국인력 제휴 문의", "사업영역 살펴보기")가 모두 스크롤 없이 보임.
- **모바일(375×812)**: 동일한 H1·서브헤드·설명·CTA 버튼 2개가 스크롤 없이 모두 보임(H1 top=201px~bottom=331px, CTA 2개 top=544~662px, 뷰포트 812px 이내). 모바일 랜딩 페이지의 이상적인 above-the-fold 패턴을 충족함.
- 헤더 CTA("제휴 문의")는 데스크톱에서 상시 노출(위치 고정), 모바일에서는 햄버거 메뉴 안에 위치.

### 내부 페이지(사업영역/회사소개/파트너십/신뢰·전문성/문의) — 개선 필요
- 5개 내부 페이지 모두 동일한 템플릿: 브레드크럼("홈 / 사업영역" 등) + 영문 아이브로우 + 영문 H1 + 국문 서브타이틀 + 설명 문단이 above-the-fold에 노출됨.
- 그러나 **모바일에서는 5개 내부 페이지 전부 above-the-fold 안에 CTA 버튼이 하나도 없음.** JS 평가 결과 fold 내 클릭 가능 요소는 로고 링크와 "홈" 브레드크럼 링크뿐이며, 실제 행동유도 버튼(제휴 문의 등)은 화면 하단 CTA 배너 또는 햄버거 메뉴를 열어야만 접근 가능. 데스크톱은 헤더에 "제휴 문의" 버튼이 상시 노출되어 문제없음.
- 즉, 모바일 트래픽 기준으로 내부 페이지 진입 시 첫 화면에서 전환 유도 요소가 전혀 보이지 않아, 스크롤을 유도해야만 문의로 이어질 수 있음.

## 2. 모바일 렌더링 이슈

### 가로 스크롤/오버플로 — 문제 없음
- 6개 페이지 × 2개 뷰포트 전수 조사 결과 `document.documentElement.scrollWidth === clientWidth` (오버플로 없음), 뷰포트보다 넓은 요소(overflow element) 0건. 가로 스크롤 발생 없음.

### 레이아웃 시프트(CLS) — 문제 없음
- 홈페이지 히어로 로드 후 6초간 `PerformanceObserver`로 layout-shift 이벤트를 측정한 결과 **CLS = 0** (엔트리 0건). 슬라이드쇼로 인한 시각적 흔들림·리플로우 없음.

### 탭 타겟(터치 영역) 크기
- 대부분의 주요 내비게이션/CTA 버튼은 40px 이상으로 양호하나, 44px 미만인 요소 다수 발견:
  - 모바일 햄버거 버튼: 40×40px (권장 44px에 근접하나 약간 부족)
  - 홈 히어로 슬라이드 인디케이터 점(dot) 3개: 32×4px — 매우 얇아 정밀 탭이 어려움(페이지네이션 점이라 스와이프로 대체 가능하지만 직접 탭하기엔 부적합)
  - 문의 폼 인풋 4개(회사명/담당자명/이메일/연락처): 277×42px — 44px 기준에 근소하게 미달
  - 개인정보 동의 체크박스: 13×13px — 기본 브라우저 체크박스 크기로 매우 작음(단, 옆 라벨 텍스트 클릭 가능 여부는 미확인)
  - 푸터 내비게이션 링크(회사소개/사업영역/파트너십/신뢰·전문성/문의): 세로 높이 16px — 텍스트 링크라 히트 영역이 얕음(다만 상하 간격 약 30px 확보)

### 가독성(폰트 크기)
- `body` 기본 폰트 16px, 본문 `<p>` 20px로 모바일에서 확대 없이 읽기에 충분함(양호).
- 단, 아래 3항에서 지적하는 저대비 소형 텍스트(10~14px, 회색 `rgb(158,158,158)`)는 크기와 대비 모두 낮아 가독성이 떨어짐.

## 3. 색상 대비(Contrast) 문제 — 신규 발견

Playwright로 실제 렌더링된 `computed style` 색상을 추출해 WCAG 명도대비를 계산함.

- `rgb(158,158,158)` 텍스트 on 흰색 배경 → **대비비 약 2.68:1** (WCAG AA 기준 일반 텍스트 4.5:1 미달)
- 해당 색상이 사용된 위치(전부 10~14px의 작은 텍스트):
  - 홈/회사소개 페이지 지표 스트립의 서브캡션: "E-9·E-7·D-2·D-4·E-8·특정기능1호", "네팔·베트남·한국·일본", "교육·송출·유학·산업·정부·지자체" (12px)
  - 사업영역/회사소개 페이지의 이미지 플레이스홀더 캡션: "[ 일본 개호·숙박 ]", "[ 매칭·계약 ]", "[ 일본 개호·숙박 현장 사진 ]" 등 (10~11px)
- 작은 글자 크기와 낮은 대비가 겹쳐 저시력 사용자·야외 밝은 화면에서 거의 안 보일 수준. WCAG AA 준수를 위해 최소 `rgb(110,110,110)` 이하(대비 ≥4.5:1)로 어둡게 하거나 폰트 크기를 키울 것을 권장.

## 4. 헤더·푸터 일관성

- **헤더**: 6개 페이지 모두 로고 + 5개 메뉴(회사소개/사업영역/파트너십/신뢰·전문성/문의) + 언어 스위처(KO/EN/JA/VI/NE) + "제휴 문의" CTA로 완전히 동일한 구조. 현재 페이지 메뉴 항목에 밑줄/하이라이트로 활성 상태 표시(예: 사업영역 페이지에서 "사업영역"에 밑줄) — 내비게이션 일관성 우수.
- **푸터**: 6개 페이지 모두 동일한 4단 구성(브랜드 소개 / MENU / COMPANY / CONTACT) + 저작권 문구 + 진출국 태그. 데스크톱·모바일 모두 100% 동일한 콘텐츠로 확인됨(`_footer_*.png` 크롭 비교) — 페이지 간 일관성 매우 우수.
- 다만 푸터 CONTACT 영역의 "전화: 입력 예정", "이메일: 입력 예정"이 6개 페이지 전부에서 노출되어, 현재 라이브 사이트에는 실제 연락처가 하나도 없는 상태(양식 제출 외 연락 수단 부재).

## 5. 콘텐츠 완성도 이슈 (플레이스홀더 노출)

라이브 프로덕션 URL 기준으로 다음이 실제 사용자에게 그대로 노출되고 있음:

1. **홈페이지 히어로 배경이 실제 사진이 아닌 대각선 줄무늬 placeholder 그라디언트**임(DOM class명 `hero-slide hero-ph`, "ph" = placeholder로 추정). 3개 슬라이드 전부 동일하게 사진 대신 줄무늬 패턴이 배경으로 사용되고 있으며, 우측 하단에 **"KV 01 · 네팔 현지 교육"** 형태의 캡션이 실제 사용자 화면에 노출됨. "KV"는 통상 "Key Visual"을 뜻하는 내부 제작 표기로 보이며, 이는 최종 사용자에게 노출되어서는 안 되는 개발/기획용 라벨로 판단됨.
2. **사업영역(/services), 회사소개(/about) 페이지의 사진 영역 6곳**이 모두 회색 줄무늬 placeholder + 대괄호 캡션(예: "[ 현지 교육 현장 사진 ]", "[ 한국 산업현장 사진 ]", "[ 일본 개호·숙박 현장 사진 ]")으로 대체되어 있어 실제 현장 사진이 전무함.
3. **파트너십(/partners) 페이지의 파트너 카드 6개 전부** 로고 자리에 "[ 로고 ]" placeholder만 노출(Richhood Overseas, Kathmandu Technical School, 청소년고용노동부, Vinako, 대한주택건설협회, 용인시). 파트너명·역할 설명은 실제 텍스트로 채워져 있어 신뢰도는 있으나, 시각적으로는 로고 없는 빈 박스 6개가 나란히 노출되어 완성도가 떨어져 보임.

CLAUDE.md/README 상 "미확정 콘텐츠는 의도적으로 입력 예정 처리" 정책이 있어 로고·사진·연락처 placeholder 자체는 알려진 이슈이나, **"KV 01 · 네팔 현지 교육" 같은 내부 라벨이 실사용자 화면에 노출되는 것은 정책과 무관한 별도의 버그성 이슈**로 보임(placeholder 사진에는 대괄호 설명 캡션만 있으면 되고, 슬라이드 순번/내부 코드명은 숨겨야 함).

## 6. 참고: 배포본과 로컬 저장소 코드 불일치 (정보성)

라이브 사이트의 실제 렌더링을 확인하는 과정에서, 로컬 저장소(`company/components/hero-slideshow.tsx`, `company/components/metrics-strip.tsx`, HEAD 커밋 `12384f5` 기준)의 코드가 라이브 사이트와 다르다는 점을 발견함:
- 로컬 HEAD의 히어로는 슬라이드쇼가 아닌 단일 정적 이미지(`/kv/redesign/hero.webp`)이며 헤드라인도 "모든 사람에게 공평한 기회를"로 다름.
- 로컬 HEAD의 지표 스트립은 "6종/**3개국**/**9곳**"인데, 라이브 사이트는 "6종/**4개국**/**6곳**"으로 수치 자체가 다름.

즉 현재 감사 대상 URL은 로컬 저장소의 최신 커밋 상태를 반영하지 못하고 있을 가능성이 높음(구버전 배포이거나, 로컬에 아직 push/deploy되지 않은 진행 중인 리디자인). 이 리포트의 발견 사항(특히 5번 항목의 placeholder/라벨 이슈) 중 일부는 이미 로컬 최신 코드에서 해결되었을 수 있으므로, 실제 개선 작업 전에 **어느 커밋이 이 Vercel 프리뷰/프로덕션에 배포되어 있는지 확인**할 것을 권장함.

## 7. 잘 되어 있는 점 (What Works)

- 6개 페이지 전수에서 가로 스크롤/오버플로 0건 — 반응형 그리드가 안정적으로 동작.
- 홈페이지 히어로 CLS 0 — 슬라이드쇼로 인한 레이아웃 흔들림 없음.
- 모바일 홈페이지 above-the-fold에 가치제안(H1+서브헤드)과 CTA 버튼 2개가 스크롤 없이 모두 노출 — 모범적인 모바일 랜딩 패턴.
- 헤더/푸터가 6개 페이지에서 픽셀 단위로 완전히 동일 — 내비게이션 일관성 우수, 현재 위치 하이라이트(밑줄) 정상 동작.
- 기본 폰트 16px(본문 20px)로 모바일 확대 없이 가독 가능.
- 모바일 햄버거 메뉴가 정상 동작하며 메뉴 내 언어 스위처에 "(준비 중)" 문구를 명시해 미완성 기능임을 사용자에게 투명하게 안내.
- CTA 배너("Let's Build Your Workforce" / "지금 채용 상담을 시작하세요")가 모든 내부 페이지 하단에 반복 배치되어, 페이지를 끝까지 읽은 사용자에게 재차 전환 기회를 제공.
- 신생 기업임을 숨기지 않고 "확인 가능한 사실만 전면에 둡니다", "검증할 수 없는 실적 수치는 표기하지 않습니다" 등 투명성 있는 카피 전략 사용.

## 8. 우선순위 권장 사항

1. (High) 히어로 슬라이드의 "KV 01 · 네팔 현지 교육" 등 내부 라벨을 프로덕션 화면에서 제거하거나 실제 사진으로 교체.
2. (High) /services, /about, /partners의 사진·로고 placeholder를 실제 자산으로 교체(최소한 라벨 텍스트의 대비/디자인을 다듬어 "준비 중" 임을 명확히 하거나, 공개 전까지 해당 섹션을 숨기는 것도 고려).
3. (Medium) 내부 페이지 모바일 above-the-fold에 CTA 버튼을 최소 1개 노출(예: 히어로 영역에도 홈페이지처럼 작은 CTA 추가).
4. (Medium) `rgb(158,158,158)` 저대비 텍스트를 WCAG AA(4.5:1) 충족하도록 어둡게 조정.
5. (Low) 문의 폼 인풋 높이(42px→44px+), 체크박스 히트 영역, 푸터 링크 히트 영역을 44px 기준에 맞게 확대.
6. (Info) 배포본과 로컬 저장소 HEAD 간 불일치 원인 확인(배포 브랜치/커밋 점검).
